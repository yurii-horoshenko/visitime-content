# VisiTime — Video Production Guide

Платформы: TikTok, Instagram Reels, YouTube Shorts (15-60 сек) + YouTube long-form (6-10 мин)
Тон: дружелюбный, data-driven, скринкасты + живое видео

---

| Штатив | Joby GorillaPod 3K или UlanziMT-08 | $30-50 |

|---------|-------------|------|

### Настройки камеры — телефон

```
Разрешение: 4K 30fps (или 1080p 60fps для slow-mo)
Экспозиция: ручной режим, выдержка 1/60
ISO: 100-400 (не выше 800, будет шум)
Баланс белого: фиксированный 5000-5600K (не авто — прыгает)
Формат: HEVC / H.265 если нужно экономить место, иначе H.264
```

### Настройки камеры — Sony / DSLR

```
Профиль: S-Log2 или S-Log3 (только если есть время на цветокоррекцию)
Для быстрого workflow: Picture Profile PP7 (HLG3) — меньше работы в монтаже
Выдержка: в 2x от FPS (30fps → 1/60, 60fps → 1/120)
Диафрагма: f/1.8-2.8 для размытия фона, f/4+ для экрана за спиной
```

### Свет

Схема для talking-head (рабочий стол / объяснение фичи):

```
[Key light] — 45° слева или справа от лица, на уровне чуть выше глаз
[Fill light] — слабее в 2x, с противоположной стороны (или белый отражатель)
[Background light] — опционально, разделяет от фона
```

Для скринкаст-стиля (лицо в углу, экран главный):
- Достаточно одного Elgato Key Light Air
- Убрать все лампы за спиной — не должно быть бликов на мониторе

### Фон

- Нейтральный серый / темно-синий — универсален для productivity-контента
- Размытый bokeh (f/1.8, 1-2м до фона) — смотрится дорого на телефоне
- Рабочий стол с деталями (ноутбук, растение, кофе) — живой b-roll стиль
- Не использовать белый фон — плохой контраст с интерфейсом приложения

## 2. Скринкасты приложения

### iOS (запись экрана)

**Встроенная запись (Control Center):**
- Настройки → Пункт управления → Запись экрана
- Запись идет в 1080p/30fps, достаточно для Reels/TikTok
- Минус: нет зума, нет подсветки тапов

**Лучшие инструменты:**

| Инструмент | Платформа | Особенности | Цена |
|-----------|----------|-------------|------|
| Record It! | iOS | Запись + редактор, аватарка в углу | Free / $9.99 |
| Rottenwood | iOS | Реакции по-над экраном | $2.99 |
| Quicktime Player | Mac + iPhone | Mirror iPhone на Mac, запись в 4K | Free |
| ScreenFlow | Mac | Профи инструмент, зум, аннотации | $129 |
| Loom | Mac/Web | Быстро + ссылка сразу | Free / $12.50/мес |

**Рекомендованный workflow для VisiTime:**
1. iPhone подключить к Mac через кабель
2. QuickTime Player → New Movie Recording → выбрать iPhone как источник
3. Запись в 1080p с Mac — чистое изображение без батареи
4. Если нужен zoom/highlight tap → ScreenFlow (наложить поверх)

### Mac (запись экрана приложения)

```
Инструмент: QuickTime (бесплатно) или ScreenFlow ($129)
Разрешение: 1080p минимум, 2K если Retina экран
FPS: 60fps — интерфейс выглядит плавнее
Cursor: использовать Cursor Pro ($10) для подсветки кликов
```

### Как делать красивый скринкаст

1. **Фрейм телефона**: вставить запись экрана в мокап телефона (Apple Device Frames в CapCut, или Rotato $9/мес)
2. **Zoom на ключевой элемент**: увеличить область кнопки/фичи в момент тапа (keyframe scale в CapCut или Premiere)
3. **Аннотации**: стрелки, подписи в CapCut или ScreenFlow
4. **Highlight tap**: мягкий круг появляется там где тап — плагин Cursor Pro (Mac) или Record It! (iOS)
5. **Чистый экран**: до записи убрать уведомления, поставить белые обои на фон за приложением, зарядить до 100%
6. **Скорость**: замедлять ключевые моменты (0.5x), ускорять навигацию (2x)

## 3. Монтаж

### Рекомендации по инструментам

| Инструмент | Лучше для | Платформа | Цена |
|-----------|----------|----------|------|
| CapCut Pro | Быстрый монтаж, TikTok/Reels, AI субтитры | Mobile + Desktop | Free / $9.99/мес |
| DaVinci Resolve | Цветокоррекция, длинный YouTube | Desktop | Free |
| Adobe Premiere Pro | Коммерческие проекты, AE интеграция | Desktop | $35/мес |
| Final Cut Pro | Mac users, скорость, ProRes | Mac only | $299 one-time |

**Для VisiTimeContent рекомендуется:**
- **60-сек TikTok/Reels**: CapCut Pro — AI субтитры + шаблоны экономят 60% времени
- **YouTube long-form (6-10 мин)**: DaVinci Resolve (бесплатно) или Final Cut Pro

### Базовый workflow для 60-секундного видео

**Этап 1 — Подготовка (5 мин)**
```
1. Создать папку: /VisiTime/Videos/YYYY-MM-DD_topic/
   - /raw — исходники
   - /screen — скринкасты
   - /export — финальные файлы
2. Просмотреть все дубли, отметить лучшие (CapCut: звездочка; Finder: тег)
```

**Этап 2 — Черновой монтаж (10 мин)**
```
1. Выложить клипы в хронологии согласно структуре (хук → проблема → решение → CTA)
2. Убрать паузы, слова-паразиты, технические сбои
3. Общая длина: целиться в 55-58 сек (оставить место для конечного экрана)
```

**Этап 3 — Детальный монтаж (15 мин)**
```
1. Скринкаст вставить поверх talking-head в нужный момент (B-roll track)
2. Добавить speed ramp: замедлить ключевой момент приложения, ускорить переходы
3. Переходы: жесткий cut (90% случаев), whip pan для смены темпа, dissolve для TIME LAPSE
4. Текстовые подписи на ключевые факты (шрифт: SF Pro / Montserrat Bold)
```

**Этап 4 — Цвет + Аудио (10 мин)**
```
Цвет:
- Применить один LUT (Kodak 2383 или VSCO A4 для теплого тона)
- Brightness +10-15 если снимали в тёмном месте
- Насыщенность -5 (немного матовее = выглядит профи)

Аудио:
- Голос: EQ High-pass 80Hz, убрать шум, легкий компрессор 3:1
- BGM: -24dB (фон, не конкурирует с голосом)
- Sound effects: свайпы, тапы по телефону +1-2 звука на скринкаст
```

**Этап 5 — Субтитры + экспорт (10 мин)**
```
- AI субтитры → ревью → стиль (см. раздел 4)
- Экспорт по платформе (см. раздел 7)
```

## 4. Субтитры

### Автосубтитры — инструменты

| Сервис | Точность | Язык | Стиль в видео | Цена |
|--------|---------|------|--------------|------|
| CapCut Pro | 95% RU/EN | RU, EN, 20+ | Да, встроенный | Free / $9.99 |
| Submagic | 97% EN | EN primary | Да, TikTok стили | $20/мес |
| Captions.ai | 95% | EN/ES | AI стили | $25/мес |
| Whisper (OpenAI) | 97% | 99 языков | Нет (только SRT) | Free (self-host) |
| Rev.com | 99% (человек) | EN | Нет | $1.50/мин |

**Workflow:**
1. CapCut → Auto Captions → выбрать язык → генерация
2. Вручную исправить: имена (VisiTime не будет правильно распознано), термины, ударения
3. Применить стиль (см. ниже)

### Стиль субтитров для TikTok / Reels

```
Шрифт: Montserrat ExtraBold или SF Pro Bold (системный на iOS)
Размер: 36-42px для 1080x1920
Цвет: Белый (#FFFFFF)
Обводка: 2-3px черная (#000000) — обязательно для читаемости на любом фоне
Тень: нет или очень мягкая
Позиция: нижняя треть экрана (y = 75-80% высоты), НЕ у самого края
```

**Стиль выделения ключевых слов (Variety style):**
```
Ключевое слово: желтый (#FFE500) или голубой (#00D4FF) текст
Размер ключевого слова: +20% от основного
Одно ключевое слово в строке — не два сразу
```

**Для YouTube Shorts:**
```
Шрифт меньше: 28-32px (YouTube автоматически масштабирует)
Можно обычный белый без выделений — аудитория старше, меньше ожиданий "варьете"
```

**Технические требования:**
- Максимум 6-7 слов в строке
- 1-2 строки одновременно (никогда не 3)
- Время показа: 1-4 сек на строку

## 5. Визуальный хук — первые 3 секунды

### Принцип

Первые 3 секунды определяют retention. Цель — остановить скролл.

Для VisiTime-контента работает:

**Паттерн A — Bold claim + экран приложения:**
```
Кадр 1 (0-0.5 сек): Крупный текст на черном фоне или анимация
  → "Я трекаю каждый час уже 90 дней"
Кадр 2 (0.5-1.5 сек): Скринкаст дашборда VisiTime с реальными данными
Кадр 3 (1.5-3 сек): Лицо в кадре с реакцией / жестом
```

**Паттерн B — Вопрос + визуальный ответ:**
```
Кадр 1 (0-1 сек): Текстовая плашка "Куда уходит твоё время?"
Кадр 2 (1-3 сек): Скринкаст с chart VisiTime — наглядный ответ
```

**Паттерн C — Результат сразу:**
```
Кадр 1 (0-3 сек): Скринкаст раздела Statistics — цифры крупно
  Поверх текст: "Вот что изменилось за месяц"
```

### Конкретные заставки для VisiTime

| Хук-идея | Визуал | Текст оверлей |
|----------|--------|--------------|
| Data reveal | Скринкаст Weekly Chart открывается | "90 дней трекинга — вот результат" |
| Before/After | Split-screen: хаос в календаре vs. чистый дашборд | "До и после VisiTime" |
| Challenge | Таймер на экране 25:00 | "Я работал без отвлечений 25 минут — вот как" |
| Surprise stat | Крупно: "4 часа 23 минуты" | "Столько я провел в TikTok на этой неделе" |
| Tutorial hook | Курсор кликает на неочевидную фичу | "Эту кнопку никто не замечает" |

### Технические параметры хука

```
Текст на экране: появляется до 0.5 сек (не позже!)
Шрифт размер: 60-80px — читается без очков
Движение: легкий zoom in (scale 100→105%) или текст slide-in слева
Музыка: BGM должна начаться с первого кадра, не после
Не использовать: черный экран в начале, логотип в начале, интро с названием канала
```

## 6. B-roll идеи для VisiTime-контента

### Категории b-roll

**Скринкасты приложения (основной b-roll)**
- Дашборд с реальными данными пользователя
- Добавление новой задачи / категории
- Таймер в работе (Pomodoro или кастомный)
- Раздел Statistics — недельный / месячный view
- Настройки / onboarding (если объясняем фичу)
- Push-уведомления от приложения

**Рабочий стол (lifestyle b-roll)**
- Ноутбук + iPhone рядом (приложение открыто на обоих)
- Руки печатают — размытый экран с VisiTime на фоне
- Кофе / блокнот рядом с телефоном где открыт таймер
- Закрытие ноутбука → телефон в руку (переход к мобильному трекингу)

**Процесс работы (abstract b-roll)**
- Таймер отсчитывает (phone screen close-up, 60fps → slow-mo)
- Галочка на задаче (close-up пальца, slow-mo)
- Свайп по экрану с переходом между разделами
- Уведомление "Session complete" на экране блокировки

**Окружение (вайб b-roll)**
- Кафе с ноутбуком (фоновый шум)
- Утро: чашка кофе + открытие приложения
- Вечер: статистика за день перед сном
- Хоум-офис: общий план + переход на крупный план экрана

### Съёмочный список для базовой библиотеки b-roll

```
Снять 1 раз, использовать многократно:

[ ] iPhone крупно — таймер работает (60fps, 10 сек)
[ ] Дашборд скролл (60fps запись экрана, 15 сек)
[ ] Добавление задачи (60fps запись экрана, 10 сек)
[ ] Statistics → Weekly view (запись экрана, 10 сек)
[ ] Руки на клавиатуре (телефон справа с VisiTime) (4K, 10 сек)
[ ] Кофе + телефон на столе (крупный план) (4K, 8 сек)
[ ] Закрытие приложения, экран блокировки с уведомлением (10 сек)
[ ] Общий план рабочего места (4K wide, 8 сек)
```

## 7. Экспорт

### TikTok

```
Разрешение: 1080 x 1920 (9:16)
FPS: 30fps
Кодек: H.264
Битрейт: 10-15 Mbps
Аудио: AAC 256kbps, стерео
Длительность: 15-60 сек (оптимально 21-34 сек для алгоритма)
Макс. размер файла: 287 МБ
```

**CapCut экспорт:** 1080p → Custom Bitrate → 10M → сохранить без лого

### Instagram Reels

```
Разрешение: 1080 x 1920 (9:16)
FPS: 30fps (60fps для smooth slow-mo)
Кодек: H.264
Битрейт: 10-15 Mbps
Аудио: AAC 256kbps
Длительность: 15-90 сек (до 15 мин в тестах, но алгоритм предпочитает до 60 сек)
Безопасная зона: top 250px и bottom 450px не должны содержать ключевых элементов (UI overlay)
```

### YouTube Shorts

```
Разрешение: 1080 x 1920 (9:16)
FPS: 30fps или 60fps
Кодек: H.264 или H.265
Битрейт: 10-20 Mbps (YouTube перекодирует, поэтому выше = лучше качество после)
Аудио: AAC 384kbps (или выше — YT справится)
Длительность: до 60 сек (ровно 60 или меньше — иначе не попадет в Shorts feed)
```

### YouTube Long-form (6-10 мин)

```
Разрешение: 1920 x 1080 (16:9) или 3840 x 2160 (4K)
FPS: 24fps (кино-стиль) или 30fps (стандарт) или 60fps (tutorial/скринкаст)
Кодек: H.264 (совместимость) или H.265 (меньше размер)
Битрейт: 1080p30 → 10-15 Mbps; 4K60 → 40-60 Mbps
Аудио: AAC 320kbps, стерео
Контейнер: MP4
```

**DaVinci Resolve экспорт (YouTube):**
```
Deliver → YouTube → выбрать 4K / 1080p
Video Quality: Restrict to 50 Mbps (4K) / 20 Mbps (1080p)
Audio: AAC 320k
```

**Final Cut Pro экспорт:**
```
File → Share → YouTube / Facebook (или Master File → H.264)
Для максимального качества: Compressor → Apple ProRes 422 → загружать ProRes прямо на YouTube
```

### Чеклист перед экспортом

```
[ ] Разрешение совпадает с целевой платформой
[ ] FPS совпадает с исходником (нет stuttering)
[ ] Субтитры не выходят за safe zone
[ ] Аудио не клипит (пик не выше -1 dBFS)
[ ] Громкость голоса -12 to -6 dBFS, BGM -24 to -18 dBFS
[ ] Финальная длительность соответствует платформе
[ ] Проиграть экспорт целиком на телефоне перед публикацией
```
