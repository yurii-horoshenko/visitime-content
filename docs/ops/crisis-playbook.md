# Crisis Playbook — что делать когда всё идёт не по плану

**Цель:** один протокол на 4 типа кризисов. Быстрая реакция без паники.

---

## Тип 1 — Внешний траур / публичное событие

Крупная трагедия в Польше / Украине / России. Публиковать beauty-контент в такой день — репутационный провал.

### Action (в течение 1 часа):

1. **Пауза всего автопостинга** — одной командой:
   ```
   останови все запланированные публикации на ближайшие 48 часов (кризис — причина: [briefly])
   ```
   Агент выключает cron / n8n flows / scheduled posts во всех соцсетях.

2. **Убрать опубликованное за последний час** — если что-то уже ушло в ленту, удаляем.

3. **Оценка**: через 24 часа пересматриваем, можно ли возобновлять. Если нет — продлеваем паузу ещё на 24.

4. **Никаких reactive-постов** от имени VisiTime («мы с вами в этот тяжёлый час») — **бренд без мнения по политическим темам**. Исключение — если прямо касается нашей аудитории (например, землетрясение в PL).

5. **После возвращения** — первый пост должен быть нейтральный (Product Education), не личный Build in Public.

## Тип 2 — Баг / downtime продукта

Приложение упало, клиенты не могут забронировать, отзывы-жалобы.

### Action (в течение 30 минут):

1. **Пауза промо-постов** (Product Education, CTA на установку) — пока не пофиксим, не зовём новых юзеров в сломанное.

2. **Сообщить в соцсети честно** — одна короткая Story в Instagram + Twitter:
   > «У нас сейчас проблема с [что именно]. Работаем над фиксом, ориентир — [время]. Если у вас не работает [функция] — не вы виноваты, это мы.»

3. **Ответы в DM** — шаблон в `content/replies/`:
   > «Спасибо что написали! Мы знаем про баг, фиксим прямо сейчас. Пришлю сообщение как только заработает.»

4. **После фикса** — пост «что произошло и как чиним». Честность тут работает сильнее чем молчание.

## Тип 3 — Публичная негативная критика / вирусный хейт

Кто-то написал жалобный пост про VisiTime, набирает просмотры.

### Action:

1. **Не удалять, не блокировать** комментатора. Это ускоряет хайп.

2. **Публичный ответ в течение 4 часов:**
   - Признать проблему если реальная: «Справедливо, мы облажались с X. Вот что делаем»
   - Объяснить без защиты если недоразумение: «Понимаем почему так могло показаться, на самом деле [факт]»
   - **Никогда:** «вы не правы», сарказм, игнор, удаление

3. **Эскалация тебе** — задача в Linear VIS-OPS-XXX с urgent статусом, ответ только после твоего апрува.

4. **После** — если урок есть, пост-рефлексия «что мы изменили после фидбэка X». Превращаем негатив в позитив.

## Тип 4 — Legal / юридическая претензия

Кто-то утверждает что мы используем его контент, имя, торговую марку без разрешения.

### Action:

1. **Ничего не отвечать** публично. Не удалять, не оправдываться.

2. **Эскалация тебе немедленно** — Telegram notification + Linear VIS-LEGAL-XXX.

3. **Временно скрыть** связанный контент (не удалить) до выяснения.

4. **Юрист или правовая консультация** прежде чем что-либо говорить.

## Kill Switch — экстренная пауза всего

Одна команда:
```
EMERGENCY PAUSE — останови всю публикацию, автопостинг, Ads, response bots
```

Действие:
- Отключается cron / n8n workflows
- Pause на всех Ad campaigns (Meta + TikTok + Google Ads)
- Community manager переключается в «только эскалация» режим
- Notification тебе «всё остановлено, жду дальнейших указаний»

**Возобновление** — только твоим явным разрешением:
```
возобнови публикацию [всю / кроме X]
```

## Blackout dates — заранее известные паузы

В `docs/ops/blackout-dates.md`:

| Дата | Причина | Что паузим |
|------|---------|-----------|
| 25.12 | Рождество | Всё кроме новогодних поздравлений |
| 01.11 | День всех святых (PL) | Юмор, скетчи — нельзя. Build in Public можно |
| 24.02 | Годовщина вторжения в UA | Пауза 48ч — чувствительная аудитория |
| 01.05 | Международный день труда | Лёгкий контент ок |

Проверяются автоматически перед публикацией — если день попадает в blackout, пост переносится на следующий доступный день.
